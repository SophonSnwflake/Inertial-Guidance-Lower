# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Embedded firmware for **STM32F411CEU6** (BlackPill board, ARM Cortex-M4 @ 100MHz). The project is named `sentry_gimbal` and is intended as an inertial guidance / gimbal control system. Running on WSL2.

## Build Commands

```bash
# Configure (Debug)
cmake --preset Debug

# Configure (Release)
cmake --preset Release

# Build
cmake --build build/Debug
# or
cmake --build build/Release

# Flash to hardware (ST-Link via OpenOCD)
./flash.sh
```

The build outputs `build/Debug/sentry_gimbal.elf` (and `.map`). A `compile_commands.json` is generated automatically for clangd.

## Flashing

`flash.sh` uses OpenOCD. In WSL2, the USB device must be forwarded from Windows first (via `usbipd` / `attach_daplink.ps1`). The script defaults to ST-Link; edit `INTERFACE_CFG` inside the script to switch to DAP-Link.

## Architecture

### Two-Layer Structure

```
TestforSTM32F411/          ← Root: user application code lives here
├── CMakeLists.txt         ← Add user sources/includes/defines here
├── CMakePresets.json      ← Debug / Release presets (Ninja + gcc-arm-none-eabi)
├── TestforSTM32F411.ioc   ← STM32CubeMX project file (re-generates CubeMX_BSP/)
├── STM32F411XX_FLASH.ld   ← Linker script (at root for user reference)
├── startup_stm32f411xe.s  ← Startup assembly (at root for user reference)
└── CubeMX_BSP/            ← AUTO-GENERATED by STM32CubeMX — treat as read-only
    ├── Core/Src/          ← main.c, gpio.c, i2c.c, spi.c, usart.c, stm32f4xx_it.c
    ├── Core/Inc/
    ├── Drivers/           ← STM32F4xx HAL + CMSIS headers/sources
    └── cmake/stm32cubemx/ ← CMakeLists.txt wiring CubeMX sources into the build
```

**Do not manually edit files inside `CubeMX_BSP/`** — they are regenerated by STM32CubeMX when the `.ioc` is re-exported. User code belongs in the `/* USER CODE BEGIN/END */` sections if editing those files is unavoidable.

### User Code Directories

```
LogicCode/                     ← Application logic (high-level control)
Task/                          ← Task implementations (e.g., RTOS tasks)
TemporarySTM32Library/
├── Algorithm/inc|src/         ← Algorithm modules (filters, kinematics, etc.)
└── Device/inc|src/            ← Custom peripheral drivers
```

`TemporarySTM32Library` is built as a separate **static library** and linked into the main target. `LogicCode/` and `Task/` sources are compiled directly into `sentry_gimbal`.

User sources and include paths are added in the **root** [CMakeLists.txt](CMakeLists.txt):

```cmake
target_sources(${CMAKE_PROJECT_NAME} PRIVATE
    LogicCode/src/foo.cpp
    Task/src/tsk_imu.cpp
)
target_include_directories(${CMAKE_PROJECT_NAME} PRIVATE
    LogicCode/inc
    Task/inc
)
```

Sources for the library go in `TemporarySTM32Library/Algorithm/src/` or `Device/src/` — they are picked up via `GLOB` in the root CMakeLists.

The `stm32cubemx` interface library (defined in `CubeMX_BSP/cmake/stm32cubemx/CMakeLists.txt`) exposes all HAL headers and `USE_HAL_DRIVER` / `STM32F411xE` macros to the project.

### Compiler Settings

C++17 is used with **no RTTI and no exceptions** (`-fno-rtti -fno-exceptions`) — keep user code exception-free. FPU is enabled (`-mfpu=fpv4-sp-d16 -mfloat-abi=hard`), so hardware float is available at no cost.

### Hardware Peripherals (from `.ioc`)

| Peripheral | Pins       | Notes                        |
|------------|------------|------------------------------|
| SPI1       | PA5/6/7    | 50 Mbps, master, full-duplex |
| I2C1       | PB6/PB7    | Standard I2C                 |
| USART1     | PA9/PA10   | Async                        |
| LED        | PC13       | GPIO output (active low)     |
| SWD        | PA13/PA14  | Debug interface              |

Clock: HSE 25 MHz → PLL (PLLM=25, PLLN=200) → **100 MHz SYSCLK**, APB1=50 MHz, APB2=100 MHz.

### Toolchain

Uses `CubeMX_BSP/cmake/gcc-arm-none-eabi.cmake`, referenced via `CMakePresets.json` at the root. The `-u _printf_float` link flag enables float formatting in `printf`. Debug builds use `-O0 -g3`; Release uses `-Os -g0`.
